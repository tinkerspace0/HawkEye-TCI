§ hotplug_callback_func

typedef void(* i3::hotplug_callback_func) (TE_STATE _teState)

function to be called when TE is arrived or removed

This type definition is for hotplug callback function which can be called if TE is arrived or removed. To call this function, user should set hotplug callback function by calling SetHotplugcallback.

§ TE_STATE

typedef struct i3::UsbState i3::TE_STATE

struct for getting TE state change when TE is arrived or removed

This structure is used for hotplug. When TE is arrived to or removed from usb port, hotplug callback function get this structure.

Apply color to input image.

Parameters
dst: output image (24bit color image, brg888)
src: input image (8bit gray image)
color: colormap to apply. its value ranges from I3_COLORMAP_WHITEHOT to I3_COLORMAP_RAINBOW.
width: width of image
height: height of image
Remarks
: This function convert 8 bit input gray image to 24 bit (bgr888) color image. Supported colormaps are whitehot, blackhot, iron, bluered, medical, purple, purpleyellow, darkblue, cyan, and rainbow.
: Since received image from Thermal Expert is 16 bit, it needs to be converted to 8 bit image before applay this function.

Connect Thermal Expert of Type B model.

Parameters
_model: Indicator for TE model. Set I3_TE_Q1 for Q1, I3_TE_V1 for V1 and I3_TE_Q2 for Q2.
hnd_dev: Device number for TE to connect
Returns
nullptr : fail to open TE.
TE_B*(not nullptr) : pointer of TE_B class
Remarks
This function get TE device handle and make TE ready to communicate with host. After connection, calibration data will be read from flash memory to create image. It may take few seconds.
This function must be called to communicate with TE.

§ ScanTE()

I3_LIB_API int i3::ScanTE ( SCANINFO *  pDevCon)

Check connected TE.

Parameters
pDevCon: Buffer to get scan results. It should MAX_USB_NUM size array.
Returns
1 : Scan successfully
Otherwise : Failed to scan
Remarks
This function scans connected TE and return the results. The result contains connected devices's ID, Product Model, and device number.
For each connected TE, a device number(0 ~ MAX_USB_NUM - 1) is assigned. TE which has device number i corresponeds to i-th element of pDevCon and its scaned result is stored in i-th element of pDevCon. This device number is used in other function as hnd_dev.
